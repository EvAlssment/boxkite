boxkite cluster-level manifests installed into namespace "{{ .Values.namespace }}".

This chart does NOT run boxkite for you. It applies the RBAC, NetworkPolicy,
and ConfigMap/Secret scaffolding a self-hosted deployment needs -- the
control plane and per-session sandbox pods are separate deployables:

  1. Deploy the control-plane API (control-plane/Dockerfile) as its own
     Deployment/Service, using ServiceAccount
     "{{ .Values.controlPlaneServiceAccountName }}" (created by ../rbac.yaml,
     not by this chart -- create it yourself or point RBAC at whatever
     ServiceAccount you actually run the control plane under).
  2. Point that deployment's RUNTIME_MODE=k8s at this cluster; SandboxManager
     creates session pods programmatically (see ../pod-template.yaml for the
     reference pod spec it builds -- not something `helm install` applies
     once).

Check what actually got applied:
  kubectl get role,rolebinding,networkpolicy,configmap,secret -n {{ .Values.namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}
